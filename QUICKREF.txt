### 🚀 5-MINUTE SETUP

1. **Create `.env.local` file**:
   ```bash
   # For local PostgreSQL:
   DATABASE_URL="postgresql://postgres:password@localhost:5432/kunafa"
   ```
   (Or use `.env.example` as template)

2. **Install dependencies** (1 min):
   ```bash
   npm install
   ```

3. **Setup database**:
   ```bash
   npm run db:push
   ```

4. **Start dev server**:
   ```bash
   npm run dev
   # → http://localhost:3000
   # Runs Next.js + WebSocket server on port 3000
   ```

5. **Login with**:
   - **Manil**: password `manil123`
   - **Ines**: password `ines123`

### 🔐 Session System

- **Storage**: localStorage (browser)
- **Data Saved**: 
  - `kunafa_token`: Session token
  - `kunafa_userId`: User identifier
  - `kunafa_connectedAt`: Connection datetime
- **Duration**: 24 hours (auto-expires & clears)
- **Backend**: Prisma Session model validates each request

---

### 💾 Database Schema

**Message Table**
```
id (PK) | sender | text | timestamp
```

**Session Table**
```
id (PK) | user | token (unique) | createdAt | expiresAt
```

### 🔌 API ENDPOINTS

**Auth**:
- `POST /api/auth/login` - Login
- WebSocket connection fails | Make sure `npm run dev` runs (not `next dev`) |
| No DATABASE_URL | Create `.env.local` with PostgreSQL URL |
| Database connection fails | Check PostgreSQL is running, check URL format |
| Build fails | `npm run db:generate && npm run build` |
| Port 3000 in use | `PORT=3001 npm run dev` |
| Session expired? | Normal - sessions expire after 24h automatically |
| Messages not real-time | Check WebSocket: Open DevTools → Inspect WebSocket tab
- `POST /api/chat/messages` - Send message
